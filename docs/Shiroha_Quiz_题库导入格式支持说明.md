# Shiroha Quiz 题库导入格式支持说明

适用范围：Shiroha Quiz 原生 Android / Compose 版题库导入解析器。

> 这份文档给用户看——告诉你可以用什么格式导入题库，每种格式的识别规则是怎样的。  
本说明整理题目文本、题号、题型、选项、答案、解析、主观题、材料题、集中答案区、Excel 表格等常见导入格式的支持情况。

解析器对多种非标准格式有兼容能力，但为了稳定识别，仍建议尽量使用”题号 + 题型 + 题干 + 选项 + 答案 + 解析”的清晰结构。

## 0. 核心导入原则

Shiroha Quiz 的默认目标不是“尽量猜”，而是先保护标准题库的稳定导入。

推荐理解为：

```text
标准格式 → 优先使用标准解析
复杂格式 → 再启用兜底候选
多个候选 → 选择题量、答案、解析和异常情况综合评分更好的结果
```

因此，题号、题型、选项、答案都很清楚的题库，应按标准结构直接导入；答案解析区、材料题、分卷重复题号、双文件答案、排版粘连等情况，才需要依赖复杂兜底策略。复杂兜底不能反过来污染正常标准解析。

## 1. 最推荐的标准格式

### 1.1 客观题标准格式

单选题：

```text
1. 【单选题】下列说法正确的是：
A. 选项一
B. 选项二
C. 选项三
D. 选项四
答案：B
解析：这里填写解析。
```

多选题：

```text
2. 【多选题】下列说法正确的有：
A. 选项一
B. 选项二
C. 选项三
D. 选项四
答案：AB
解析：这里填写解析。
```

判断题：

```text
3. 【判断题】公共服务应当坚持便民原则。
答案：正确
解析：判断题解析。
```

### 1.2 主观题标准格式

```text
4. 【简答题】请简述提升公共服务质量的路径。
参考要点：
1. 完善服务流程；
2. 加强人员培训；
3. 建立反馈机制。
解析：本题应从流程、人员、反馈等方面作答。
```

### 1.3 最稳妥的题型写法

最推荐这两类：

```text
1. 【单选题】题干内容
1. 单选题 题干内容
```

章节标题也推荐使用：

```text
一、单选题
二、判断题
三、简答题
```

## 2. 支持的题号格式

## 2.1 数字题号

支持：

```text
1. 题干
1、题干
1．题干
1: 题干
1：题干
1) 题干
1）题干
第1题. 题干
第1题：题干
第 1 题：题干
```

也支持题号和题干之间使用空格：

```text
1 题干内容
```

较紧凑写法也可识别：

```text
1题干内容
```

建议：紧凑写法更适合 1–2 位题号；题号较长时，建议使用 `1. 题干`、`1、题干`、`第1题：题干` 这类更清晰的格式。

### 2.2 方括号题号

支持：

```text
【1】题干
[1] 题干
```

### 2.3 公考面试 / 主观题题号

支持：

```text
问题一：题干
问题1：题干
题目一：题干
第一问：题干
第一个问题：题干
第一道题：题干
第1问：题干
第1题：题干
```

也支持部分中文编号题号：

```text
一、请谈谈你的看法。
二、请分析原因。
三、请提出对策。
```

注意：如果中文编号明显是章节标题，例如 `一、标准客观题测试区`、`二、判断题`，解析器会优先按章节处理，不会当作题目。

## 3. 支持的题型标记

题型标记既可以写在单题题干前，也可以写成章节标题，让后续题目继承该章节题型。

### 3.1 支持的题型名称

单选类：

```text
单选题
单选
单项选择题
单项选择
选择题
```

多选类：

```text
多选题
多选
多项选择题
多项选择
不定项选择题
不定项选择
```

判断类：

```text
判断题
判断
是非题
是非
对错题
对错
```

填空类：

```text
填空题
填空
补全题
补全
```

主观题类：

```text
简答题
简答
问答题
问答
面试题
面试
结构化面试题
结构化面试
公考面试题
公考面试
公务员面试题
公务员面试
名词解释
论述题
论述
材料分析题
材料分析
案例分析题
案例分析
综合题
综合
```

### 3.2 单题内题型标记

支持以下单题写法：

```text
1. 【单选题】题干内容
1. [单选题] 题干内容
1. （单选题）题干内容
1. (单选题) 题干内容
1. 〔单选题〕题干内容
1. 〖单选题〗题干内容
1. 《单选题》题干内容
1. 单选题 题干内容
```

其他题型同理：

```text
2. [多选题] 题干内容
3. （判断题）题干内容
4. (填空题) 题干内容
5. 〔简答题〕题干内容
6. 〖问答题〗题干内容
7. 《材料分析题》题干内容
8. 综合题 题干内容
```

解析器会尝试把题型标签从题干中移除，并将其作为该题的题型依据。

### 3.3 章节题型标记

支持通过章节标题给后续题目指定题型：

```text
一、单选题
二、多选题
三、判断题
四、填空题
五、简答题
六、公考面试题
七、材料分析题
```

也支持章节标题中的括号题型：

```text
一、【单选题】
二、[多选题]
三、（判断题）
四、(填空题)
五、〔简答题〕
六、〖问答题〗
七、《材料分析题》
```

章节标题本身不会被当作题目；后续没有单独题型标签的题目，会优先继承该章节题型。

### 3.4 题型标签后的分隔符

题型标签后可以直接接题干，也可以使用空格或常见分隔符：

```text
1. 单选题 题干内容
1. 单选题：题干内容
1. 单选题-题干内容
1. 单选题—题干内容
1. 【单选题】题干内容
1. [单选题] 题干内容
```

## 4. 支持的章节标题

以下章节标题会被识别为分区标题，不会误当作题目：

```text
第一部分 常识判断
第二部分 言语理解
第三部分 数量关系
第四部分 判断推理
第五部分 资料分析
```

也支持：

```text
一、常识判断
二、言语理解
三、数量关系
四、判断推理
五、资料分析
```

常见知识模块类标题也可识别：

```text
一、公共基础知识
二、专业知识
三、安全知识
四、理论知识
五、综合能力
六、结构化面试
七、公考面试
```

测试文档或整理文档中的说明性章节也会尽量避免被误识别为题目，例如：

```text
一、标准客观题测试区
二、格式变体与边界测试区
三、主观题测试区
四、材料题与多问测试区
五、集中答案区测试样本
六、极端格式测试样本
```

## 5. 支持的选项格式

支持 A–G 选项，大小写不敏感，导入后会统一为大写。

### 5.1 独立行选项

支持：

```text
A. 选项内容
B. 选项内容
C. 选项内容
D. 选项内容
```

也支持：

```text
A、选项内容
A．选项内容
A: 选项内容
A：选项内容
A) 选项内容
A）选项内容
```

### 5.2 同一行多个选项

支持：

```text
A. 选项一 B. 选项二 C. 选项三 D. 选项四
A、选项一 B、选项二 C、选项三 D、选项四
A. 选项一；B. 选项二；C. 选项三；D. 选项四
```

### 5.3 选项续行

如果选项已经开始，后续非题号、非答案、非解析的普通文本行，可能会被追加到上一选项后面。

示例：

```text
A. 这是一个较长选项
需要换行继续说明
B. 第二个选项
```

### 5.4 不推荐的选项格式

当前主要支持 A–G，不建议使用 H 之后的选项：

```text
H. 选项
I. 选项
```

当前也不建议把甲乙丙丁作为标准选项标记：

```text
甲. 选项
乙. 选项
丙. 选项
丁. 选项
```

建议改为：

```text
A. 选项
B. 选项
C. 选项
D. 选项
```

## 6. 支持的答案标签

### 6.1 客观题答案标签

支持：

```text
答案
正确答案
参考答案
标准答案
答
```

标准写法：

```text
答案：B
正确答案：B
参考答案：B
标准答案：B
答：B
```

### 6.2 主观题答案标签

主观题还支持：

```text
参考要点
参考思路
答题要点
答题思路
作答思路
评分要点
参考作答
```

示例：

```text
参考要点：围绕问题背景、原因分析、解决措施作答。
答题思路：先概括问题，再分析原因，最后提出对策。
参考作答：我会从以下几个方面处理……
```

### 6.3 方括号答案标签

支持：

```text
【答案】B
【答案】：B
【答案：B】
[答案] B
[答案：B]
【正确答案】B
【参考答案】B
[正确答案] B
[参考答案] B
```

## 7. 支持的答案分隔符

答案标签和答案值之间支持多种分隔符。

以下写法等价于 `答案：B`：

```text
答案 B
答案：B
答案:B
答案,B
答案，B
答案、B
答案.B
答案．B
答案;B
答案；B
```

同样适用于：

```text
正确答案 B
参考答案，B
标准答案；B
答 B
```

## 8. 支持的括号答案格式

支持选择题、判断题答案放在半角或全角括号中。

### 8.1 题干末尾括号答案

支持：

```text
1. 下列说法正确的是（B）
A. 选项一
B. 选项二
C. 选项三
D. 选项四
```

也支持：

```text
题干(B)
题干（B）
题干(AB)
题干（AB）
题干（正确）
题干（错误）
题干（对）
题干（错）
题干（是）
题干（否）
题干（√）
题干（×）
题干（True）
题干（False）
```

解析器会尝试把括号中的答案提取出来，并从题干中移除。

### 8.2 答案行括号答案

支持：

```text
答案（B）
答案(B)
答案：（B）
答案：(B)
答案（AB）
答案：（AB）
正确答案（B）
参考答案（B）
标准答案（B）
【答案】（B）
[答案] (B)
```

### 8.3 集中答案区括号答案

支持：

```text
1.（B）
2.(AB)
3. 答案（C）
4. 答案：(D)
5. 正确答案（正确）
6. 参考答案（错误）
```

## 9. 支持的客观题答案值

### 9.1 单选答案

支持：

```text
A
B
C
D
E
F
G
```

大小写不敏感：

```text
a
b
c
```

### 9.2 多选答案

支持紧凑写法：

```text
AB
ABC
ABCD
ABCDEFG
```

支持分隔符写法：

```text
A B
A、B
A，B
A,B
A;B
A；B
A/B
A\B
```

导入后会解析为多个选项答案，并去重。

### 9.3 判断题答案

支持：

```text
正确
错误
对
错
是
否
√
×
True
False
T
F
X
```

导入规则：

```text
正确 / 对 / 是 / √ / True / T → A
错误 / 错 / 否 / × / False / F / X → B
```

如果判断题没有显式选项，解析器会自动补：

```text
A. 正确
B. 错误
```

## 10. 支持的解析格式

支持以下解析标签：

```text
解析
答案解析
说明
解题思路
```

标准写法：

```text
解析：这里是解析。
答案解析：这里是解析。
说明：这里是说明。
解题思路：这里是解题思路。
```

方括号写法：

```text
【解析】这里是解析。
[解析] 这里是解析。
【答案解析】这里是解析。
[答案解析] 这里是解析。
```

答案和解析可以在同一行：

```text
答案：B 解析：这里是解析。
答案 B 解析：这里是解析。
答案（C）解析：这里是解析。
```

## 11. 支持的主观题 / 面试题格式

### 11.1 参考要点格式

支持：

```text
1. 【简答题】请简述提升公共服务质量的路径。
参考要点：
1. 完善服务流程；
2. 加强人员培训；
3. 建立反馈机制。
解析：本题应从流程、人员、反馈等方面作答。
```

支持的主观答案标签：

```text
参考要点
参考思路
答题要点
答题思路
作答思路
评分要点
参考作答
答案
参考答案
```

主观题、填空题的答案会按文本答案处理。请不要把带顿号、逗号、分号的参考答案拆成选择题答案。

例如：

```text
参考答案：检查设备、核对参数、进行安全交底。
```

应整体保留为文本答案，而不是拆成多个选择题选项或多个客观题答案。

### 11.2 答题思路格式

支持：

```text
2. 【面试题】群众情绪激动到窗口反映问题，你怎么办？
答题思路：
首先安抚情绪，其次核实问题，再依法解释并推动解决。
参考作答：
我会先稳定群众情绪，认真倾听诉求……
```

### 11.3 面试多问格式

支持：

```text
问题一：请概括材料反映的问题。
参考要点：……

问题二：请提出改进建议。
参考要点：……
```

也支持：

```text
第一问：……
第一个问题：……
第一道题：……
第1问：……
第1题：……
```

## 12. 支持的集中答案区

```text
集中答案
集中解析

标准答案
正确答案
答案解析
答案与解析
答案及解析
试题答案
答题要点
参考要点
参考思路
答题思路
作答思路
评分要点
答案区
解析区
答案部分
```

集中答案区不会被当作题目本体，会单独用于匹配题号答案。

### 12.1 一行一道答案

支持：

```text
1. A
2. B
3. 正确
4. ABC
```

也支持：

```text
第1题 A
第2题：B
1、A
1．A
1: A
1：A
```

### 12.2 带答案标签

支持：

```text
1. 答案 A
1. 答案：A
1. 答案，A
1. 答案；A
1. 答案（A）
1. 正确答案（B）
1. 参考答案：AB
```

### 12.3 带解析

支持：

```text
1. A 解析：这里是解析。
2. 答案：B 解析：这里是解析。
3. 答案（C）解析：这里是解析。
```

### 12.4 一行多个答案

支持：

```text
1. A 2. B 3. C 4. D
1、A 2、B 3、C 4、D
1. 答案 A 2. 答案 B
1【答案】A 2【答案】B
```

### 12.5 范围答案

支持：

```text
1-5: A B C D A
1~5: A B C D A
1～5: A B C D A
1至5: A B C D A
```

建议：范围答案后面的答案值使用空格分隔，稳定性更高。

### 12.6 集中答案区中的题型标题

集中答案区中也可以通过章节标题指定后续答案类型：

```text
一、单选题
1. A
2. B

二、判断题
3. 正确
4. 错误
```

题型标题也支持括号写法：

```text
一、[单选题]
1. A
2. B

二、（判断题）
3. 正确
4. 错误
```

## 14. 支持的材料题格式

当前支持材料题、共用题干后的问题识别。建议材料引导行单独成行，后续每道小题继续使用清晰题号。

### 14.1 材料分析题

```text
材料一：
这里是一段材料文本。

45. 【材料分析题】根据材料，概括主要问题。
参考要点：……

46. 【材料分析题】根据材料，提出改进建议。
参考要点：……
```

### 14.2 共用题干选择题

```text
根据以下材料，回答 1-2 题：
这里是一段材料文本。

1. 下列说法正确的是？
A. 选项一
B. 选项二
答案：A

2. 根据材料，下列做法正确的有？
A. 选项一
B. 选项二
答案：AB
```

也可以和集中答案解析区配合：

```text
根据以下材料，回答 1-2 题：
这里是一段材料文本。

1. 下列说法正确的是？
A. 选项一
B. 选项二

2. 根据材料，下列做法正确的有？
A. 选项一
B. 选项二

答案解析
1.【答案】A【解析】第一题解析。
2.【答案】AB【解析】第二题解析。
```

注意：

```text
材料一：
材料二：
```

这类材料引导行不会被当作单独题目。

边界规则：

- 材料引导行应相对独立，例如 `根据以下材料，回答 1-2 题：`。
- 小题仍建议使用 `1.`、`2.`、`第1题` 等明确题号。
- 如果一行已经是正式题目，例如 `1. 根据材料回答1-2题，下列说法正确的是？`，解析器会优先按普通题目处理，不会把整行当成材料引导行。
- 材料题属于复杂兜底能力，导入后请重点核对题干、答案和解析是否对应。

对于材料高度混排、题目与答案交错严重的文档，建议先使用 AI 重构，再进行人工核对。

## 16. 不建议或暂不稳定的格式

### 16.1 选项超过 G

当前选项主要支持：

```text
A–G
```

不建议使用：

```text
H. 选项
I. 选项
```

### 16.2 使用甲乙丙丁作为选项

暂不作为标准支持格式：

```text
甲. 选项
乙. 选项
丙. 选项
丁. 选项
```

建议改成：

```text
A. 选项
B. 选项
C. 选项
D. 选项
```

### 16.3 没有题号的长篇连续题库

无题号题目有一定兜底识别能力，但稳定性低于有题号格式。

建议每道题都带题号。

### 16.4 集中答案区没有题号

不推荐：

```text
A
B
C
D
```

推荐：

```text
1. A
2. B
3. C
4. D
```

### 16.5 材料原文和问题高度混排

例如材料中夹杂多个未编号问题、答案区和解析区反复穿插，当前仍可能需要 AI 重构辅助。

建议流程：

```text
AI重构 → AI核对 → AI解析 → 人工确认写入
```

## 19. 编写题库时的建议

1. 每道题尽量写清楚题号，例如 `1. 题干`。
2. 客观题建议显式写题型，例如 `【单选题】`、`【多选题】`、`【判断题】`。
3. 选项建议使用 A–D 或 A–G，不建议使用甲乙丙丁。
4. 答案建议使用 `答案：B`、`答案：AB`、`答案：正确`。
5. 解析建议使用 `解析：` 单独成行。
6. 集中答案区一定要带题号，例如 `1. A`、`2. B`。
7. 主观题建议使用 `参考要点：`、`答题思路：`、`参考作答：`。
8. 材料题建议材料和题目分开写，问题最好单独编号。

## 20. Excel 表格导入（v0.3.7 新增）

原生版支持直接导入 `.xlsx` / `.xls` 文件。解析器会自动识别表格中的列，提取题干、选项、答案和解析。

### 20.1 推荐表格格式

| 列                   | 说明           |
| ------------------- | ------------ |
| 题干                  | 必填           |
| 选项A                 | 单选题/多选题/判断题  |
| 选项B                 | 同上           |
| 选项C / D / E / F / G | 根据需要         |
| 答案                  | 必填，如 A、AB、正确 |
| 解析                  | 可选           |
| 章节 / 分区             | 可选，分卷重复题号时建议填写 |

### 20.2 注意事项

- 表格第一行应作为表头
- 每行一道题
- 判断题答案写"正确"或"错误"
- 如果同一份表格按章节、题型分区，并且每个分区题号从 1 重新开始，建议增加“章节 / 分区”列，或在表格中保留清晰的章节标题。
- 表格答案不建议只写一串无题号答案；如需集中答案表，最好包含题号、答案、章节 / 分区信息。
- 格式非常混乱、题目被吞并或答案区难以匹配时，建议先用 AI 重构，再人工核对。

## 21. JSON 题库导入（v0.7.4 新增）

Shiroha Quiz 原生版支持从多个 JSON 格式导入题库。

### 21.1 支持格式总览

| 格式 | 说明 | 导入入口 |
|------|------|----------|
| 多题库数组 | `[{"name":"题库1","questions":[...]}]` | 导入备份 JSON |
| 单题库对象 | `{"name":"题库","questions":[...]}` | 导入备份 JSON |
| 题目数组 | `[{"type":"single","question":"...","answer":["A"]}]` | 普通导入或备份导入 |
| Web 备份包 | `{"app":"Shiroha Quiz","banks":[...]}` | 导入备份 JSON |
| 原生备份包 | `{"kind":"shiroha_quiz","banks":[...]}` | 导入备份 JSON |
| 对象选项 | `{"options":{"A":"选项文本","B":"..."}}` | 任意 JSON 导入 |

### 21.2 答案解析字段

支持以下答案字段（优先级从高到低）：

- `answer`（数组 `["A","B"]` 或字符串 `"A,B"`）
- `answers`、`correctAnswer`、`correctAnswers`

答案字符串支持分隔符：空格、逗号、分号、斜杠、竖线。

### 21.3 选项字段

选项支持两种格式：

**数组格式**：
```json
{"options": [{"key": "A", "text": "选项内容"}, ...]}
```

数组内元素可以是对象或纯字符串。纯字符串会按 A/B/C/D 顺序自动生成 key。

**对象格式**（v0.7.4 新增）：
```json
{"options": {"A": "选项内容", "B": "选项内容"}}
```

对象 key 即选项标号，value 可以是字符串或 `{"text":"..."}` 对象。

### 21.4 扩展元数据字段

每道题可附带云端字段（导入时自动保留）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `subject` | 字符串 | 学科 |
| `grade` | 字符串 | 年级 |
| `difficulty` | 字符串 | 难度 |
| `knowledgePoints` | 字符串数组 | 知识点 |
| `tags` | 字符串数组 | 标签 |
| `source` | 字符串 | 题目来源 |
| `sourceFileId` | 字符串 | 源文件 ID |
| `version` | 整数 | 题目版本号 |
| `reviewStatus` | 字符串 | 审核状态 |
| `aiConfidence` | 数字 | AI 置信度 |
| `warnings` | 字符串数组 | 警告信息 |

### 21.5 Web 备份导入

从 Web 版导出的备份包含错题本、收藏夹、记录和设置。原生端导入 Web 备份时：

- ✅ 题库内容正常导入
- ❌ 错题本、收藏夹、练习记录、设置不恢复（Web/原生数据结构不同）
- 导入完成后提示"仅导入题库"

### 21.6 多题库处理

- 普通导入页检测到多题库 JSON 时，会引导用户到"设置/导出 → 导入备份 JSON"入口
- 备份导入页支持一次性导入多个题库
