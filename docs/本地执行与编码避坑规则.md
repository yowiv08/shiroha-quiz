# 本地执行与编码避坑规则

这份规则用于减少在 Windows 本地开发时反复遇到的：

- 中文路径乱码
- PowerShell / CMD / Bash 来回切换
- Gradle 缓存锁
- Kotlin daemon 权限提示
- Python 处理中文图片路径失败

## 一、默认约定

1. 默认 shell：`PowerShell`
2. 默认文本编码判断：
   - 源码 / Markdown / JSON / XML：`UTF-8`
   - 用户题库导入文本：先 `UTF-8`，再 `GB18030`
3. 默认路径写法：
   - 绝对路径
   - PowerShell 中用单引号

## 二、最容易踩坑的错误

### 1. 控制台乱码，不等于文件损坏

PowerShell 里看到中文乱码时，优先判断为**显示层乱码**，不要立刻回写文件。

推荐做法：

- 用 Python 直接读文件内容核对
- 用 `Select-String` 检查关键文本
- 必要时做 `unicode_escape` 输出

### 2. 中文路径被外部工具误读

Python、Pillow、部分命令行工具对中文路径不稳定时：

- 不要硬顶着原路径处理
- 先复制到 ASCII 临时路径
- 处理完再复制回目标路径

### 3. Gradle wrapper 联网 / 锁文件 / 权限问题

优先顺序：

1. 使用本地 `gradle-8.7`
2. 使用项目内 `.gradle-user-home`
3. 尽量 `--offline`
4. 优先复用工作区已有 jar 和缓存

### 4. Kotlin daemon 报错但任务未必失败

如果出现：

- `Could not connect to Kotlin compile daemon`
- `AccessDeniedException ... kotlin-daemon-client`

不要立刻判定整次构建无效。

先继续看：

- 是否 fallback 成功
- 是否真有编译错误
- 是否真有测试失败

## 三、以后执行时的固定流程

### 文本 / 代码检查

1. 先 PowerShell 读目录、确认真实路径
2. 再修改文本
3. 如果出现中文乱码，先做内容核对，不直接回写

### 图片 / 二进制处理

1. 中文路径图片先复制到 ASCII 临时路径
2. 再用 Python / Pillow 处理
3. 再写回目标位置

### Android 原生测试

1. 优先本地 Gradle
2. 优先项目内缓存
3. 优先离线
4. 缺测试依赖时优先找本地 jar
5. 真失败才修代码，不因 daemon 提示误判

## 四、当前项目的额外要求

- 用户明确要求：**原生安卓优先**
- 当用户要求“不要动 Web / 不要动别的版本”时，只允许修改：
  - `apps/android/**`

这条必须先检查，再执行。
