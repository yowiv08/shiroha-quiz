# 下阶段开发计划

当前优先级以原生 Compose 版稳定化为主。WebView 壳版继续维护，但不是当前主要开发线。

## P0：发布前稳定性

- 修复原生页面残留文案问题。
- 继续核校真实题库导入解析结果，重点关注题干清洗、答案区合并、双文件导入和乱码文件读取。
- 确认练习、考试、错题、记录主流程稳定。
- 补齐离线测试依赖或测试说明，避免本地未缓存依赖时误判测试失败。
- 发布前统一检查版本号、包名、图标、开屏图和导出 APK 文件名。

## P1：原生体验收口

- 优化题库详情、导入预览和题库复查页面的信息层级。
- 完善填空题 / 简答题作答体验。
- 优化 AI 核对 / AI 解析的错误提示和失败恢复。
- 继续整理首页、考试结果页和空状态的素材使用。
- 补充发布截图和简短使用说明。

## P2：跨端与长期整理

- 确认 Web 备份 JSON 与原生数据结构的迁移边界。
- 继续沉淀 parser 规则样例，保持 Web 版和原生版导入结果可对照。
- 视情况整理 Room 持久化、批量导入和更大题库的性能方案。
- Web 壳保留为稳定对照版本，后续只做必要维护。
