# Shiroha Quiz 后续功能开发计划 33-34

当前版本：原生 `v0.7.6-native`，Web `v0.6.0-alpha`。原生 Compose 版为主开发线，Web 端同步演进解析架构。

---

## 一、33 阶段前已完成能力清单

以下能力已落地，后续阶段不得回退：

### 1. 导入解析

- 解析器重构：选项检测逻辑集中管理（`CompactQuestionRepair`），精简 `StandardQuestionParser`
- 主观题误判防御：`looksLikeSubjectivePrompt`、`looksLikeMisparsedObjectiveQuestion`，防止简答/填空被误判为选择题
- 英文单词内字母误判修复（`looksLikeOptionMarkerInsideAsciiWord`、`looksLikeDottedEnglishAbbreviation`）
- 双文件合并顺序修复：各候选先合并答案再比较，避免提前淘汰
- JSON 导入全面兼容：多题库数组、题目数组、单题库对象、Web 备份包、key-value 对象选项
- 答案解析字段兼容：`answer`/`answers`/`correctAnswer`/`correctAnswers` 多别名
- 图片标记保护：选项拆分不误入图片标记内部
- 回归测试 12/12（待修复 07_compact_format）

### 2. Web 解析架构（V34）

- 原文区间局部修复（v58.9.10）：漏题补回、粘题拆分、基于源文本字节位置追踪
- 过度拆题局部合并（v58.9.11）：真实题号锚点间合并多余题目
- 答案解析区分区感知匹配（v58.9.11）：卷/章/分组三级匹配权重
- 局部修复守卫与审计增强（v58.9.12）：序列对齐验证、富内容保留、连续性评分收紧
- 英文题干勘误（v58.9.8.2）：`isBareEnglishStemStartV5982`，A/a 开头句子不误判为选项
- APP_VERSION V34：标准解析与局部修复版

### 3. 练习与考试

- 选项打乱（练习+考试独立控制）：固定 A/B/C/D 标签，仅随机内容映射
- 考试选项打乱：独立的 `ExamOptionShuffle` 开关，同一场考试内顺序固定
- AI 单题分析：练习提交后或背题模式下可触发，AI 独立判断参考答案并与题库对照
- 编辑器 AI 补解析：题库编辑/审阅/导入预览/快速编辑四个入口统一集成 `AiAnalysisFillPanel`

### 4. 错题本与管理

- 错题本作用域切换：当前题库 / 全部题库，影响列表、清空、首页统计、智能复习范围
- 记录筛选：记录列表与详情均支持"全部/只看错题"筛选
- 导出流程分离：题库管理页导出只含题库，设置页处理完整备份

### 5. 后台化

- M1-M7 完成：用户认证（bcrypt+JWT）、题库/任务/作答/统计表、文件上传、管理面板、使用端 API、用户组发布、回传幂等
- 服务端 `normalizeOption` 支持对象格式、`extractBanksFromImport` 兼容 Web 备份包

### 6. 工程

- 清理未启用的 monorepo 骨架文件（pnpm-workspace.yaml / package.json / packages/）
- 回归测试 build.gradle 补充 `assets/` 和 `util/` 源目录
- Question 云端字段完整序列化（subject、grade、difficulty 等 11 字段）

## 二、当前阶段定位

33 阶段结束后，原生 Compose 版已具备完整的导入→练习→考试→错题→复习→AI 辅助闭环，Web 端解析架构大幅升级。阶段 33-34 的核心任务：

5. **工程基础设施建设**：测试体系、性能优化

## 四、Web 端维护策略

Web 端当前定位：

- 解析架构已升级至 V34，与原生端共享核心解析规则
- 继续作为在线体验入口（GitHub Pages）和 WebView 壳基础
- 功能开发以原生版为主，Web 版只做解析兼容和跨端数据互通
- 两端题库格式保持兼容，不做破坏性变更

## 六、开发约束

1. **默认开发范围** — 原生 `apps/android/app/src/native/`，Web `apps/web/`
2. **最小修改原则** — 不顺手重构，不引入无关变更
3. **不回退既有功能** — 已完成能力清单中的所有功能不得被破坏
4. **标准解析优先** — 标准题库解析主线必须稳定，复杂兜底不能污染标准格式
5. **后台化本地提交** — 后台模块（admin/server）只在 `content-admin` 分支本地提交，不推送远端
6. **敏感信息隔离** — 代码、测试样例、文档中不得写入真实组织、人名等敏感信息
7. **标签保护** — 永远不删除 Git 标签，只删除 Draft 草稿 Release
