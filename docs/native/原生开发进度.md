# Shiroha Quiz 原生 Android 开发进度

> 更新日期：2026-06-21  
> 本文只记录原生 Android Compose 版进度。Web 版、历史 WebView 壳和旧计划不在本文展开。

## 当前定位

原生 Android Compose 版是 Shiroha Quiz 当前主推安装包和主要开发线。

| 项目 | 当前状态 |
| --- | --- |
| 技术路线 | Kotlin + Jetpack Compose + Material3 |
| 包名 | `com.reqir.shirohaquiz` |
| 发布线 | `v0.8.x-native` |
| 当前阶段 | 功能已较完整，重点转向解析稳定性、刷题体验、回归测试和真实题库兼容 |

当前原生版已经可以完成完整学习闭环：

```text
导入题库
↓
预览核对
↓
练习 / 考试 / 背题
↓
错题复盘 / 收藏 / 斩题 / 智能复习
↓
记录回看
↓
备份导出与跨端恢复
```

## 已落地能力

### 题库导入与解析

- 支持文本、`.docx`、`.xlsx` / `.xls`、`.txt`、普通题库 JSON 和文字层 PDF 辅助导入。
- 支持双文件导入：题目文件和答案文件分开时，可按题号、章节、顺序和候选评分合并。
- 支持标准题库解析：题号、题干、选项、答案、解析、题型、分区 / 分卷识别。
- 支持表格题库解析：按列识别题号、题型、题干、选项、答案和解析。
- 支持集中答案解析区：可将 `1.【答案】A【解析】...` 这类答案区合并回对应题目。
- 支持共用题干 / 材料题兜底识别。
- 支持多空填空题：显式空位、括号空位、分区继承和答案逐空拆分。
- 支持 docx 内嵌图片提取与题目绑定。
- 支持普通题库 JSON、Web 导出 JSON、原生 ZIP 备份和图片题字段兼容。
- 支持导入预览、异常提示、题型/答案/题干手动修正和全文查找替换。

### 刷题与考试

- 支持普通练习、随机练习、顺序练习、即时反馈、自动下一题和批量提交。
- 支持背题模式：直接显示答案和解析，不计入普通练习记录。
- 支持斩题功能：将已掌握题移出普通练习池，并可在题库详情中恢复。
- 支持题目收藏、快速编辑、字号调整和紧凑选项显示。
- 支持顺序练习进度记忆，退出时可保存当前位置。
- 支持考试模式：题量、分值、倒计时、答题卡、交卷提醒和结果统计。
- 支持单选、多选、判断、填空、多空填空和简答题。

### 错题、记录与复习

- 练习和考试错题自动进入错题本。
- 支持错题按当前题库 / 全部题库切换作用域。
- 支持错题掌握状态、连续答对规则和手动标记掌握。
- 支持智能复习，根据错题状态提示到期复习。
- 支持刷题记录列表、记录详情和只看错题筛选。

### AI 辅助能力

- 导入页支持 AI 核对题型、答案和解析。
- 导入页支持 AI 生成或补全解析。
- 编辑器、题库审阅、导入预览和快速编辑中集成 AI 补解析入口。
- 练习页支持 AI 单题分析，可对照题库答案给出置信度和人工确认提示。
- 支持 DeepSeek、OpenAI 兼容接口和自定义接口配置。
- AI 结果仅作为辅助参考，不自动覆盖题库答案。

### 数据与跨端互通

- 支持题库、错题、收藏、记录和设置的本地保存。
- 支持 JSON 备份导出与恢复。
- 支持原生 ZIP 备份携带图片素材。
- 支持 Web 端与原生端题库备份互导，兼容题型大小写和图片字段差异。

### UI 与体验

- 使用 Compose + Material3 原生界面。
- 支持浅色 / 暗夜模式。
- 支持平板侧边导航，宽屏体验优于单纯手机布局放大。
- 支持 Shiroha 模式：统一控制角色插画、开屏图和应用图标相关视觉元素。
- 支持系统返回键映射到二级页面返回。
- 已建立设计 Token、通用卡片、按钮、状态标签和页面组件规范。

## 工程与测试

- 原生源码位于 `apps/android/app/src/native/`。
- 导入解析核心位于 `apps/android/app/src/native/java/com/yiqiu/shirohaquiz/importer/`。
- 外部回归测试位于 `test/native-parser-regression/`。
- 回归测试通过 `manifest.json` 管理样例，不直接改 App 源码。
- 当前正在继续补充真实题库边界样例，尤其是主观题答案延续、复杂答案区、章节重复题号和表格答案合并等场景。

## 近期收口重点

1. 继续收口原生解析器真实题库边界，优先保证标准格式不被复杂兜底误伤。
2. 完善外部回归样例，新增用例必须能说明具体风险，不能为了覆盖率硬塞失败样例。
3. 继续优化导入预览和异常提示，让用户更容易判断是格式问题、答案缺失还是解析器边界。
4. 收口大题库性能、备份可靠性和图片题资源管理。
5. 继续打磨平板/手机两端刷题体验，保持原生 Android 交互优先。
