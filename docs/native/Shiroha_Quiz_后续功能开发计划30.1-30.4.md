# Shiroha Quiz 后续功能分级规划

## 1. 规划目标

本规划用于梳理 Shiroha Quiz 后续三个体验类功能的开发顺序：

1. 页面切换动画
2. 开屏图片 / 启动页
3. 夜间模式

这三项功能都属于体验优化，但影响范围不同。为了避免一次性改动过大，建议分阶段推进。

推荐路线：

> 先做导航切换动画 → 再加开屏图片 → 再抽主题变量 → 最后加夜间模式

这样每一步都能单独测试，出问题也方便回退。

---

## 2. 总体开发原则

### 2.1 保持最小必要修改

每个版本只解决一个核心问题，不要顺手重构、顺手改布局、顺手加新功能。

### 2.2 不扩大修改范围

当前规划主要针对 Shiroha Quiz 的移动端体验优化。  
具体改 Web 版、WebView 壳，还是原生 Android，需要以后每次任务中单独确认。

### 2.3 优先保证性能

移动端和 WebView 环境要避免过重动画：

- 少用大面积动态 blur
- 少用复杂阴影
- 少用长时间弹簧动画
- 避免页面切换时整页大幅移动
- 避免启动页图片过大拖慢加载

### 2.4 保持项目视觉统一

整体视觉方向保持：

- 蓝白基调
- 清爽、明亮
- 卡片化
- 云母 / 亚克力 / 液态玻璃质感
- 不要过度厚重
- 不要大面积深蓝或刺眼蓝色

## 3. 阶段一：页面切换动画

### 3.1 目标

优化点击底部导航栏后的页面切换体验，让页面不再生硬切换，同时保持轻量、干净、不卡顿。

### 3.2 推荐效果

底部导航切页采用：

- 页面内容淡入
- 轻微上浮
- 短时长
- 不做大幅左右滑动
- 不使用重 blur
- 不影响刷题和考试页面性能

推荐视觉感受：

> 页面不是“滑过去”，而是内容轻轻浮现。

### 3.3 推荐参数

```text
时长：160ms – 200ms
透明度：0 → 1
位移：translateY(6px - 8px) → translateY(0)
缓动：ease-out / cubic-bezier(0.22, 1, 0.36, 1)
```

### 3.4 导航栏反馈

点击底部导航时，导航栏本身可以有轻微反馈：

- 选中项浅色胶囊背景出现
- 图标轻微放大到 1.04 – 1.06
- 文字颜色从灰蓝变为品牌蓝
- 点击阴影立即结束，不要残留
- 动画控制在 120ms – 160ms

### 3.5 不推荐效果

不建议使用：

- 大幅左右滑动
- 整页缩放
- 复杂弹簧动画
- 长时间残留阴影
- 强毛玻璃动态模糊
- 页面整体上下大幅滑动
- 翻页式动画

这些效果容易让 App 显得花哨，也可能影响移动端性能。

### 3.6 建议版本

```text
v30.1：导航切换动效优化
```

### 3.7 修改范围

只改：

- 页面切换动画
- 导航点击反馈
- 按钮点击反馈时长

不改：

- 题库逻辑
- 导入逻辑
- 数据结构
- 页面整体布局
- 夜间模式
- 开屏页

## 4. 阶段二：加入开屏图片 / 启动页

### 4.1 目标

新增 App 启动时的开屏页，让项目更像完整 App，同时保持启动速度和移动端体验。

### 4.2 开屏页视觉方向

开屏页应保持：

- 手机全面屏竖屏比例
- 蓝白 / 浅色基调
- 云母 / 亚克力 / 液态玻璃质感
- 主视觉为角色学习画面
- 桌面元素保持统一
- 不裁切成横图或局部图
- 不做过深、过灰、过蓝的背景

### 4.3 推荐形式

优先使用：

```text
静态 WebP / PNG + CSS 动画
```

不建议一开始使用 GIF，因为：

- GIF 文件体积容易过大
- 清晰度通常不如静态图
- 移动端加载压力更高
- 不利于后续压缩和适配

更好的方式是：

- 图片本身保持静态高清
- 用 CSS 做轻微淡入
- 用 CSS 做进度条或微光动画
- 退出时做淡出

### 4.4 启动逻辑

建议启动流程：

```text
App 启动
显示开屏页
初始化题库 / 本地状态
至少显示 1.2 秒
加载完成后淡出
进入首页
```

开屏页主要承担视觉缓冲，不建议让它承载复杂加载逻辑。

### 4.5 显示时间

建议显示时间：

```text
建议：1.2 – 1.8 秒
最长不超过：2 秒左右
```

时间太短会看不清，时间太长会影响进入 App 的效率。

### 4.6 图片资源建议

| 用途 | 格式 | 说明 |
|---|---|---|
| 原始备份 | PNG | 保留高清原图 |
| App 实际使用 | WebP | 控制体积，提升加载速度 |
| 宣传展示 | PNG / JPG | 用于 B 站、GitHub、社交平台 |

### 4.7 体积建议

```text
宽度：1080px 或 1440px
比例：手机全面屏竖屏
格式：WebP
体积：尽量控制在 500KB – 1.5MB
```

不要直接把 4K 原图塞进 App，否则可能拖慢启动速度。

### 4.8 开屏页内容建议

如果图片本身已经包含标题和加载条，页面里不要再重复叠加太多文字。

推荐结构：

```text
全面屏背景图
Shiroha Quiz 标识
一句短标语
轻量加载条 / 微光动画
```

如果图片已经自带这些元素，则代码层只负责显示和淡入淡出。

### 4.9 建议版本

```text
v30.2：新增开屏页与启动动画
```

### 4.10 修改范围

只改：

- 新增开屏页
- 开屏图资源
- 启动淡入淡出动画
- 开屏页全面屏适配
- 图片资源压缩

不改：

- 首页结构
- 题库逻辑
- 导入逻辑
- 夜间模式
- 页面切换动画之外的其他动效

## 5. 阶段三：整理主题变量

### 5.1 目标

在正式加入夜间模式前，先整理浅色主题变量。

夜间模式不要直接到处硬改颜色，而应该先把颜色集中管理。

### 5.2 为什么要先做主题变量

如果不先整理变量，直接做夜间模式，容易出现：

- 有些页面颜色没改到
- 有些文字在夜间模式下看不清
- 卡片、弹窗、按钮颜色不统一
- 后续维护困难
- 改一个颜色需要全局搜索很多文件

所以建议先做一个过渡版本：  
先把浅色模式的颜色集中成变量，但暂时不开放夜间模式入口。

### 5.3 建议整理的变量

```text
--bg-app
--bg-card
--bg-elevated
--text-main
--text-sub
--border-soft
--brand-primary
--brand-primary-soft
--nav-bg
--nav-active
--button-primary
--button-secondary
--input-bg
--danger
--success
--warning
--shadow-soft
```

### 5.4 重点要求

这个阶段不急着做夜间模式按钮。

先保证：

- 现有浅色模式视觉不变
- 各页面颜色来源统一
- 后续切换夜间模式时不用逐页硬改
- 不破坏现有 UI

### 5.5 建议版本

```text
v30.3：主题变量整理
```

### 5.6 修改范围

只改：

- CSS 颜色变量
- 现有页面颜色引用方式
- 少量必要的类名或样式引用

不改：

- 页面布局
- 功能逻辑
- 夜间模式入口
- 本地存储逻辑
- 题库数据
- 导入解析逻辑

## 6. 阶段四：加入夜间模式

### 6.1 目标

新增夜间模式，让 App 支持暗色外观，同时保持 Shiroha Quiz 原本的蓝白、玻璃、卡片化气质。

### 6.2 夜间模式风格

不建议做纯黑模式。

Shiroha Quiz 更适合：

```text
深蓝灰背景
半透明深色卡片
浅蓝高亮
低亮度白色文字
柔和边框
克制阴影
```

这样可以避免画面太沉，同时保持项目的蓝白气质。

### 6.3 推荐入口

夜间模式入口建议放在：

```text
设置页 / 我的页
```

不建议放在：

- 首页
- 底部导航
- 刷题页面主操作区
- 考试页面主操作区

夜间模式属于全局设置，不应该干扰核心刷题流程。

### 6.4 切换方式

基础版可以先做两个选项：

```text
浅色模式
夜间模式
```

后续再扩展为：

```text
跟随系统
浅色模式
夜间模式
```

### 6.5 存储逻辑

用户选择后保存到本地，下次打开自动沿用。

推荐逻辑：

```text
用户选择主题
保存到 localStorage
下次启动读取主题
应用对应 class / data-theme
```

### 6.6 夜间模式重点检查页面

夜间模式不能只看首页，需要逐页检查：

```text
首页
题库导入
题库管理
练习模式
考试模式
错题本
练习记录
设置页
弹窗 / 更多菜单
导入结果提示
图片题显示
按钮和输入区域
```

### 6.7 容易出问题的地方

重点检查：

- 题目文字可读性
- 选项卡片边框
- 正确 / 错误颜色
- 弹窗背景
- 底部导航选中态
- 错题本空状态
- 导入结果提示
- 表格 / 列表边界
- 图片题背景
- 文件选择区域
- 输入框占位文字
- 禁用按钮颜色
- 练习和考试页面的状态标签

### 6.8 建议版本

```text
v30.4：新增夜间模式
```

### 6.9 修改范围

只改：

- 夜间模式变量
- 设置页切换入口
- 本地保存主题设置
- 全页面暗色适配

不改：

- 题库解析逻辑
- 练习 / 考试业务逻辑
- 数据结构
- 无关 UI 重构
- 新增无关功能

## 8. 测试重点

### 8.1 页面切换动画测试

检查：

- 点击底部导航是否顺滑
- 页面切换是否有卡顿
- 动画是否过长
- 是否出现点击阴影残留
- 是否影响刷题和考试页面
- 低性能手机上是否仍然流畅

### 8.2 开屏页测试

检查：

- 是否保持全面屏比例
- 是否出现裁切
- 是否加载过慢
- 是否能正常淡出
- 是否会卡在开屏页
- 横竖屏或不同屏幕比例下是否正常
- 图片体积是否过大

### 8.3 主题变量测试

检查：

- 浅色模式是否保持原样
- 首页、导入页、练习页、考试页颜色是否统一
- 按钮、卡片、弹窗是否出现样式异常
- 是否有硬编码颜色遗漏

### 8.4 夜间模式测试

检查：

- 全页面文字是否清晰
- 选项卡片是否可读
- 正确 / 错误状态是否明确
- 弹窗和底部导航是否协调
- 图片题背景是否突兀
- 文件导入区域是否可辨认
- 设置页切换是否保存成功
- 下次启动是否沿用用户选择

---

## 9. 最终建议

当前最稳的开发顺序是：

```text
1. 导航切换动画
2. 开屏图片
3. 主题变量整理
4. 夜间模式
```

不要把页面切换动画、开屏图片和夜间模式合并成一次大更新。

每个版本只解决一个核心问题：

```text
v30.1 解决切换生硬
v30.2 解决启动视觉
v30.3 解决主题基础
v30.4 解决夜间模式
```

这样更符合当前项目的开发原则：

- 最小必要修改
- 不扩大范围
- 不影响题库逻辑
- 方便测试
- 方便回退
- 避免一次改崩
