# Shiroha Quiz 原生 Android / Compose 版 30.5～31 开发计划

> 说明：本文是阶段性规划 / 历史设计记录，部分版本号和待办项可能已经被后续实现覆盖。  
> 当前项目状态请优先参考根目录 `README.md`、`CHANGELOG.md` 和 `docs/native/原生开发进度.md`。

## 一、阶段目标

30.5～31 阶段主要围绕 **刷题体验、练习效率、阅读体验和个人偏好设置** 进行优化。

本阶段重点不是新增大型题库能力，而是把已有练习 / 考试流程做得更顺手，目标包括：

```text
减少刷题点击次数
降低长题干和长解析的滑动负担
支持更灵活的题量选择
让常用练习设置可以记住
提供单选 / 判断快速刷题能力
逐步引入批量做题后统一提交的模式
```

整体策略是：

```text
先做低风险 UI 与交互优化
再做个人偏好和设置持久化
最后再做批量做题这类练习状态模型改造
```

---

## 二、开发范围与原则

本阶段默认只处理原生 Android / Compose 版：

```text
apps/android/app/src/native/
```

除非明确要求，不改：

```text
apps/web/
apps/android/app/src/main/assets/web/
```

默认不改：

```text
题库数据结构
导入解析核心逻辑
Web 版
WebView 壳内 assets/web
```

继续坚持：

1. 默认交付最小替换包。
2. 只打包实际修改文件。
3. 不回退前面已确认功能。
4. 不写入真实内部信息。
5. 不默认完整 Android 编译验证。
6. 练习状态模型相关改动单独成阶段，不和普通 UI 小优化混在一起。

# 三、30.5 计划：刷题 / 考试页面基础体验优化

## 1. 目标

30.5 主要做低风险体验优化，不改变答题记录、错题本、练习完成逻辑。

重点解决：

```text
刷题时切题不够快
题干和选项占用空间偏大
解析展示不够清楚
题量选择不够灵活
```

---

## 2. 手势切题

### 2.1 功能目标

在练习页和考试页增加横向滑动切题能力，减少频繁点击上一题 / 下一题按钮。

建议先实现：

```text
左滑：上一题
右滑：下一题
```

后续如果实际使用中觉得不符合移动端常见习惯，再统一调整为：

```text
左滑：下一题
右滑：上一题
```

### 2.2 交互建议

手势不要加在整个页面最外层，避免影响上下滑动阅读。

建议加在题目主体卡片或题目内容区域。

触发规则建议：

```text
横向滑动距离超过阈值才触发
横向滑动明显大于纵向滑动才触发
第一题不能继续上一题
最后一题不能继续下一题
练习页仍要遵守原有“是否允许下一题”的规则
```

### 2.3 个人偏好设置

建议在：

```text
我的 / 设置页 → 个人偏好 → 刷题体验
```

增加开关：

```text
启用滑动切题
```

说明文案：

```text
在练习页和考试页左右滑动切换题目。
```

默认值建议：

```text
开启
```

如果后续用户反馈误触较多，可以改为默认关闭，或增加滑动方向设置。

---

## 3. 题量选择增加“自定义题量”

### 3.1 功能目标

题量不再只依赖固定数量，允许用户按本次练习需要输入题数。

建议练习页题量选项：

```text
10题
15题
自定义
50题
100题
一半
全部
```

建议考试页题量选项：

```text
10题
15题
自定义
50题
100题
全部
```

原先固定的 `20题` 不再作为主要固定入口，改为：

```text
自定义
```

### 3.2 自定义题量交互

点击“自定义”后弹出轻量输入弹窗：

```text
自定义题量
请输入本次练习题数
范围：1～当前可用题数
```

行为建议：

```text
只允许输入正整数
最小值为 1
最大值不超过当前筛选后可用题数
超过范围时自动修正或提示
确认后使用该题量开始练习 / 考试
```

### 3.3 题量不足时的显示规则

如果当前可用题量不足，应自动隐藏不合适的固定选项。

例如当前只有 12 题：

```text
10题
自定义
全部12题
```

当前只有 8 题：

```text
自定义
全部8题
```

自定义入口建议保留，因为用户可能只想练 3 题、5 题。

---

## 4. 题干与选项显示更紧凑

### 4.1 功能目标

降低小屏手机上阅读题干、选项、解析时的滑动频率。

### 4.2 调整建议

只调整练习页和考试页，不全局修改组件字号。

建议方向：

```text
题干字号略微降低
题干行高略微降低
选项文字略微降低
选项卡片上下内边距略微缩小
选项之间间距略微缩小
```

注意不要压缩过度，仍要保证：

```text
暗夜模式可读
长题干可读
多选题选项不拥挤
小屏设备不显得压迫
```

---

## 5. 解析多行展示优化

### 5.1 功能目标

把解析从“解析：一整段文字”改成更适合阅读的结构。

建议展示为：

```text
解析

本题考查……

A项，……
B项，……
C项，……
D项，……
```

### 5.2 展示规则

建议只做展示层处理，不改题库数据。

规则：

```text
“解析”作为单独标题
解析正文单独显示
保留原始换行
如果原解析没有换行，但包含 A项 / B项 / C项 / D项，可在这些标记前自动补换行
解析为空时沿用当前缺解析提示
```

---

## 6. 30.5 验收标准

```text
练习页支持滑动切题
考试页支持滑动切题
滑动切题不明显影响上下滚动
题量支持 10题 / 15题 / 自定义
固定 20题不再作为主要入口
自定义题量不能超过当前可用题数
题干和选项更紧凑但仍清楚
解析标题和正文分离
原有练习 / 考试流程不被破坏
```

# 四、30.6 计划：个人偏好与练习设置记忆

## 1. 目标

30.6 重点处理 **长期行为偏好** 和 **练习设置持久化**。

核心原则：

```text
练习页放本次练习要临时选择的东西
个人偏好放长期习惯、开关、默认行为
```

因此：

```text
本次题量、本次题型、本次组题方式 → 留在练习页
是否记住设置、是否自动下一题、是否启用滑动切题 → 放到个人偏好
```

---

## 2. 个人偏好结构调整建议

建议在：

```text
我的 / 设置页 → 个人偏好
```

中把相关内容整理为：

```text
外观设置
启动设置
刷题体验
```

其中“刷题体验”用于放长期行为开关。

建议结构：

```text
刷题体验

记住上次练习设置
单选 / 判断自动下一题
启用滑动切题
默认练习模式
```

如果暂时不做“默认练习模式”，可以先隐藏，等批量做题模式完成后再加。

---

## 3. 记住上次练习设置

### 3.1 功能目标

用户上次选择的练习配置，下次进入练习页时自动恢复。

建议记住：

```text
上次题量类型
上次自定义题量
上次选择的题型范围
上次选择的组题方式
```

### 3.2 个人偏好开关

开关名称：

```text
记住上次练习设置
```

说明文案：

```text
进入练习页时自动恢复上次选择的题型、题量和组题方式。
```

默认值建议：

```text
开启
```

### 3.3 行为建议

```text
用户修改题量、题型或组题方式后自动保存
下次进入练习页自动恢复
如果当前题库没有上次选择的题型，则自动过滤
如果上次自定义题量超过当前可用题数，则自动降到当前最大可用题数
如果关闭该开关，则进入练习页使用默认设置
```

---

## 4. 单选 / 判断自动下一题

### 4.1 功能目标

减少单选题和判断题的刷题点击次数。

### 4.2 个人偏好开关

开关名称：

```text
单选 / 判断自动下一题
```

说明文案：

```text
选择单选题或判断题选项后，自动提交并进入下一题。
```

默认值建议：

```text
关闭
```

原因：

```text
避免改变老用户习惯
避免用户误点后立即提交
避免用户想看即时解析时被自动跳走
```

### 4.3 行为建议

单选题：

```text
点击选项 → 记录答案 → 自动提交 → 短暂延迟 → 下一题
```

判断题：

```text
点击选项 → 记录答案 → 自动提交 → 短暂延迟 → 下一题
```

多选题：

```text
不自动下一题
```

建议延迟：

```text
150～250ms
```

最后一题：

```text
自动提交后进入完成状态，不再调用下一题
```

---

## 5. 启用滑动切题

### 5.1 功能目标

给手势切题提供长期偏好开关，避免用户不习惯时无法关闭。

### 5.2 个人偏好开关

开关名称：

```text
启用滑动切题
```

说明文案：

```text
在练习页和考试页左右滑动切换题目。
```

默认值建议：

```text
开启
```

如果后续误触反馈较多，可以改为默认关闭。

---

## 6. 默认练习模式

该设置建议等批量做题模式完成后再启用。

开关 / 选项名称：

```text
默认练习模式
```

选项：

```text

```

默认值：

```text
即时反馈
```

说明：

```text
即时反馈：每题提交后查看结果和解析。

```

---

## 7. 30.6 验收标准

```text
个人偏好中有独立“刷题体验”区域
记住上次练习设置开关可用
关闭记住设置后不会自动恢复上次配置

```

# 五、30.8 计划：批量做题后统一提交

## 1. 目标

30.8 引入新的练习模式：

```text
批量做题
```

该模式适合连续刷题：

```text
先做完一组题
过程中不逐题显示答案
点击提交本组后统一判分
再按题号顺序查看答案和解析
```

这一步会改变练习状态模型，必须单独做，不建议和 30.5 / 30.6 的轻量优化混在一起。

---

## 2. 练习模式

保留当前模式：

```text
即时反馈
```

新增模式：

```text
批量做题
```

默认模式建议仍为：

```text
即时反馈
```

避免影响现有用户习惯。

---

## 3. 状态模型建议

建议新增或扩展练习状态：

```text
practiceMode
practiceDraftAnswers
practiceBatchSubmitted
```

含义：

```text
practiceMode：当前练习模式，即时反馈 / 批量做题
practiceDraftAnswers：批量做题时每道题的临时答案
practiceBatchSubmitted：当前批量练习是否已经提交
```

实际字段命名按现有 `QuizRepository.kt` 风格处理。

---

## 4. 批量做题流程

未提交前：

```text
进入练习
选择答案
下一题
继续选择答案
完成本组
点击提交本组
```

提交时：

```text
统一计算每道题对错
生成练习记录
错误题写入错题本
正确题更新掌握状态
进入解析复盘状态
```

提交后：

```text
从第 1 题开始查看解析
可上一题 / 下一题复盘
可结束练习
可再来一组
```

---

## 5. UI 建议

即时反馈模式保持当前按钮逻辑：

```text
提交答案 / 查看解析
上一题
下一题
```

批量做题模式未提交前：

```text
上一题
下一题
提交本组
```

批量做题模式已提交后：

```text
上一题解析
下一题解析
结束练习
再来一组
```

顶部状态建议显示：

```text
批量做题中：已答 6 / 10
```

如果提交时仍有未答题，建议提示：

```text
还有 2 题未作答，是否仍然提交？
```

---

## 6. 与个人偏好的关系

批量做题模式完成后，可以在：

```text
我的 / 设置页 → 个人偏好 → 刷题体验
```

启用：

```text
默认练习模式
```

选项：

```text
即时反馈
批量做题
```

用户仍然可以在练习页临时切换本次模式。

---

## 7. 记录和错题本规则

批量做题模式下，不应在用户选择答案时立即写入错题本或练习记录。

正确规则：

```text
用户点击“提交本组”后统一计算并写入
```

提交前：

```text
只保存临时答案
不显示正确答案
不显示解析
不写错题本
不写练习记录
```

提交后：

```text
显示正确答案
显示解析
写入错题本
写入练习记录
更新掌握状态
```

---

## 8. 30.8 验收标准

```text
即时反馈模式仍可正常使用
批量做题模式可连续做完一组题
未提交前不会显示正确答案和解析
未提交前不会写入错题本
未提交前不会生成练习记录
提交本组后统一判分
提交后可逐题查看答案和解析
错题本和练习记录结果正确
未答题提交时有提示或明确处理逻辑
默认练习模式可在个人偏好中设置
```

# 六、31.0 计划：刷题体验收口与复盘优化

## 1. 目标

31.0 用于对刷题体验进行收口和细节打磨。

重点方向：

```text
答题卡状态更清晰
错题复盘更顺畅
手势方向最终确认
批量做题模式体验完善
```

---

## 2. 答题卡状态增强

批量做题模式稳定后，可以增强答题卡状态。

未提交前：

```text
未答
已答
当前题
```

提交后：

```text
正确
错误
当前题
收藏
```

这样用户能快速知道：

```text
哪些题没做
哪些题已经答过
哪些题答错
当前复盘到哪里
```

---

## 3. 只看错题复盘

批量提交后，可以增加：

```text
只看错题
查看全部
```

用途：

```text
用户提交一组题后，快速只复盘错误题。
```

注意：

```text
只看错题只是本次练习结果过滤
不应改变正式错题本数据
要保留返回全部题目的入口
```

---

## 4. 手势方向确认

30.5 阶段如果先按 issue 原始诉求实现：

```text
左滑：上一题
右滑：下一题
```

31.0 可根据实际使用反馈决定是否调整为更常见的移动端逻辑：

```text
左滑：下一题
右滑：上一题
```

如果分歧明显，可以在个人偏好中增加：

```text
滑动方向
```

选项：

```text
左滑下一题
左滑上一题
```

但不建议一开始就加入，避免设置项过多。

---

## 5. 快速刷题入口整理

如果 30.6 和 30.8 的功能都稳定，可以把练习模式进一步整理为：

```text
即时反馈
快速刷题
批量做题
```

含义：

```text
即时反馈：每题提交后查看结果和解析

```

这一项属于产品结构调整，建议最后再做。

## 6. 31.0 验收标准

```text
30.5～30.8 的功能不回退
答题卡能区分未答、已答、正确、错误、当前题
批量提交后可只看错题
只看错题不影响正式错题本
手势方向符合最终使用习惯
个人偏好设置不显得拥挤
练习页本身仍保持清爽
```

# 七、最终推荐执行顺序

## 第一阶段：30.5

先做：

```text
练习页 / 考试页手势切题
10题 / 15题 / 自定义题量
题干和选项显示压缩
解析多行展示
```

不做：

```text
自动下一题
批量统一提交
练习设置持久化
错题复盘重构
```

原因：

```text
风险低
改动集中
用户感知明显
便于快速验证
```

---

## 第二阶段：30.6

继续做：

```text
个人偏好新增“刷题体验”
记住上次练习设置
记住自定义题量
单选 / 判断自动下一题
启用滑动切题开关
```

原因：

