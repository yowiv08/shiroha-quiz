# 安卓组件与状态说明

## 目的

这份文档的作用，是把我们前面已经定下来的页面想法，拆成真正能落开发的单元。

内容包括：

- 通用 UI 组件
- 关键页面状态
- 核心用户流程

## 通用组件

### 1. 页面头部块

用于：

- 首页
- 练习设置
- 导入页
- 我的页面

内容：

- 英文或功能短标签
- 页面标题
- 页面说明文字

规则：

- 标题永远是视觉焦点
- 说明文字只讲当前页的目的，不重复标题

### 2. 毛玻璃卡片

用于：

- 首页主卡
- 指标卡片
- 题目卡
- 导入结果卡

规则：

- 圆角要大
- 背景半透明偏白
- 描边柔和
- 不使用过重阴影

### 3. 标签胶囊

用于：

- 题型
- 进度
- 当前题库状态
- 导入方式
- 选中状态

规则：

- 文案尽量短
- 一行里不要堆太多，超过 3 到 4 个就要合理换行

### 4. 操作按钮胶囊

用于：

- 主要 CTA
- 次要操作

样式：

- 实心主按钮
- 轻描边副按钮

规则：

- 主操作永远放前面
- 小屏幕一行尽量不超过 2 个主要按钮

### 5. 选项卡片

用于：

- 单选题
- 多选题
- 判断题

状态：

- 默认
- 已选中
- 答对
- 答错
- 禁用

规则：

- 已选中状态必须非常明显
- 提交答案后，正确/错误状态要压过单纯选中状态

### 6. 上传面板

用于：

- 导入页

状态：

- 默认
- 已选文件
- 正在识别
- 识别失败
- 识别成功

### 7. 提示卡

用于：

- 警告
- 导入异常
- 考试提醒

样式类型：

- 中性
- 警告
- 成功
- 危险

### 8. 统一按压反馈（ShirohaClick）

用于替代原生 Compose `clickable` 的所有可点击组件。

规则：

- 全局统一按压水波纹样式
- 取代之前的 `shirohaNoRippleClickable`，改用标准 Compose 交互

### 9. 图片展示组件（QuestionImageComponents）

用于题目详情、预览和核对中的图片渲染。

规则：

- 自适应宽度，限制最大高度
- 暗夜模式下背景透明无边框
- 导入预览中尺寸更紧凑

## Shiroha 模式与双图标

### Shiroha 模式

统一开关，控制所有角色相关视觉元素：

- 开屏图（Splash Screen）
- 各页面角色插画（IllustrationHeroCard）
- 应用启动器图标（双图标切换）

关闭后：角色插画不渲染，开屏图跳过，启动器切为简洁默认图标。
面向场景：可在更通用的场景下使用（二次元向）。

### 双图标系统

| 别名 | 图标 | 默认状态 |
|------|------|---------|
| `DefaultLauncher` | `ic_launcher_default` | 启用 |
| `ShirohaLauncher` | `ic_launcher_shiroha` | 禁用 |

- 运行时通过 `LauncherIconSwitcher` 切换，不杀进程
- 两套图标均在 `native/` 源集内独立管理

## 页面状态模型

### 首页状态

需要的数据：

- 当前题库摘要
- 总题数
- 错题数
- 最近成绩
- 最近使用题库

界面状态：

- 只有默认示例题库
- 已切换到用户导入题库
- 还没有刷题记录
- 正常回访用户

### 练习设置页状态

需要的数据：

- 当前题库
- 练习来源
- 顺序模式
- 题量
- 分类过滤

状态：

- 初始状态
- 因题库为空无法开始
- 已准备好，可以开始

### 练习过程状态

需要的数据：

- 题目列表
- 当前索引
- 当前选择答案
- 是否已提交
- 正确数
- 错误数

状态：

- 加载中
- 作答中
- 已提交
- 已完成

关键规则：

- 未提交时，只显示选择状态
- 提交后，才显示对错和解析
- 下一步操作必须永远清楚

### 导入页状态

需要的数据：

- 导入模式
- 已选文件
- 粘贴文本
- 解析策略
- 识别结果
- 警告列表

状态：

- 空状态
- 文件已选
- 正在识别
- 识别成功
- 识别成功但有异常
- 识别失败

### 导入预览状态

需要的数据：

- 识别总数
- 题型统计
- 异常数量
- 过滤后的异常列表

状态：

- 未显示
- 正常预览
- 异常优先预览
- 正在编辑某题

### 考试页状态

需要的数据：

- 考试配置
- 生成后的试卷
- 倒计时
- 用户答案
- 提交结果

状态：

- 配置中
- 考试中
- 已交卷
- 超时结束

## 核心用户流程

### 流程 A：首次打开

1. 打开 App
2. 看到默认 C1 示例题库
3. 点开始刷题
4. 做几道题
5. 看到解析和记录

### 流程 B：导入新题库

1. 打开导入页
2. 选择单文件或双文件
3. 选择文件
4. 识别结果预览
5. 必要时修正异常题
6. 确认导入
7. 设为当前题库
8. 进入练习

### 流程 C：错题复习

1. 用户在练习或考试中答错
2. 错题自动进入错题本
3. 用户从首页或我的页面进入错题复习
4. 只练未掌握错题

## 导航结构

底部 4 个入口：

- 首页
- 练习
- 导入
- 我的

内部页面流转：

- 首页 -> 题库切换
- 首页 -> 错题复习
- 练习 -> 练习设置
- 练习 -> 正式刷题
- 练习 -> 考试设置
- 练习 -> 正式考试
- 导入 -> 导入预览
- 导入 -> 单题修正
- 我的 -> 记录
- 我的 -> 导出
- 我的 -> 设置
