# Shiroha Quiz 原生 Android / Compose 视觉规范

> 说明：本文是原生 Compose 视觉规范文档，文件名中的 v30 是设计阶段代号，不等同于 App 发布版本号。  
> 文档内容承接 v25 之后的移动端优化经验，当前实现状态以根目录 `README.md` 和 `docs/native/原生开发进度.md` 为准。  
> 适用范围：`apps/android/app/src/native/`  
> 目的：作为后续原生 Android / Compose UI 调整的统一视觉标准，避免每个页面各自使用不同的尺寸、间距、字体和动效参数。

## 5. 页面头部规范

| 项目                | 标准                                        |
| ----------------- | -----------------------------------------:|
| 页面结构              | `kicker + title + hero card`              |
| kicker 到 title 间距 | `8dp`                                     |
| title 到 Hero 卡片间距 | `16dp`                                    |
| Hero 卡片到下一内容卡片间距  | `16dp`                                    |
| kicker 样式         | `labelLarge / 13sp / Bold / BrandPrimary` |
| title 样式          | `displaySmall / 34sp / SemiBold`          |
| 页面外层间距            | `horizontal 20dp, vertical 8dp`           |

### 5.1 页面说明文字规则

页面头部下方不放额外长说明文字。

说明文字应放入以下位置：

- Hero 卡片内部；
- 内容卡片内部；
- 独立提示卡片内部。

不要让说明文字破坏 `title → Hero card` 的标准间距。

## 13. 页面级统一规则

| 项目                 | 标准                      |
| ------------------ | ----------------------- |
| 首页、练习页、导入页、错题本、设置页 | 使用统一页面边距和一级区块间距         |
| Hero 卡片            | 高度、图片尺寸、图片动效一致          |
| 内容卡片标题             | 统一 `titleLarge`         |
| 同排按钮               | 高度、权重、视觉尺寸一致            |
| 页面说明文字             | 不破坏头部标准间距               |
| 小卡片图片              | 使用统一显示框尺寸               |
| 后续新增 UI            | 优先复用现有 Token，不新增随意 dp 值 |

---

## 14. 后续修改约束

后续修改必须遵守：

1. 基于最新有效包继续叠加。
2. 不能覆盖、回退前面已确认有效的修改。
3. 每次只改当前明确问题。
4. 不顺手重构。
5. 不乱动业务逻辑。
6. 不改 Web 版。
7. 不改 WebView 壳。
8. 原生版只动：

```text
apps/android/app/src/native/
```
